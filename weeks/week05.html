Types Of Errors

System error: A problem with the System Programmer error: When the programmer makes the mistakes User error: When the user enters wrong data.

Exceptions

It is also a type of error. This error returns a message or value so we have a clue in where to look for the error.

Stack Traces

When an exception happens, a stack trace is produced. This happens when we have several functions being called by other functions. We need to go up in the tree to look for the problem.

Warnings

The warnings appears when the error is not that bad. The program will not crash.

Strict Mode

This is an option to avoid big errors. It helps avoiding "poor style," this reduces the posibility to write bad code. We apply it by using "Use strict."

Linting Tools

These are tools to test Javacript's quality. It highlights "sloppy" programming. It doesn't mean the code will be perfect but it will be more consistent.

Feature Detection

This is the recommended way to determine browser support. When using it, we apply an if statement to check if the method exists.

The Trusty Alert

alert() is used for debugging, this stops the programn when reachs alert() until OK is clicked.

Debugging Tools

Browser allow us to debugg in the web, like Firefox, Edge, Chrome, Safari.

Error Objects

It can be created with a constructor function, and it helps to identify errors. ner Error(), EvalError, RangeError, ReferenceError, SyntaxError, TypeError, URIError, InternalError. We also create objects as constructors to create customized errors. They also have properties. Name, that returns the name of the error constructor funtion. message, that returns the description of the error. Stack, that provides the stack error.

Throwing Exceptions

The throw statement can used to make our own exceptions. The best practice is to used it with error. For example: throw new Error().

Exception Handling

A better way of handling errors is to catch them. We can deal with the errors or ignore them if we want to. We can use try, catch and finally to deal with errors.

Tests

Tests are important in programming, our code will be more likely to work better and avoid future problems when we test it. There is a way of writing code called Test-driven Development. When we write tests, then code to pass that test, then we make better the code (code refactor), and then refactor the code. This is called Test-driven Development.
